# Screenshot checklist

Required screenshots for the Devpost submission and README. Capture each on the **local run** (so AI replies are real) at standard browser zoom, with no DevTools panel open.

## Required screenshots

- [ ] **Home** — landing screen with offshift check-in entry and guided flow tiles
- [ ] **Offshift check-in** — chat flow showing a reflective AI reply (no real personal content)
- [ ] **Moral Injury walkthrough** — one of the 5 Shay/Litz steps with the prompt visible
- [ ] **Sleep support** — nightly sleep check with grounding prompt
- [ ] **ProQOL-5** — screener mid-flow (a question) and the result framing screen
- [ ] **Shadow Logbook** — saved entry from a completed walkthrough
- [ ] **Privacy** — privacy / settings screen showing local-storage and encryption controls

## Capture conventions

- **Resolution:** 1440×900 minimum. Retina captures preferred.
- **Format:** PNG. Lossless.
- **Aspect:** keep the full app frame visible — do not crop the crisis footer out.
- **Theme:** use the default theme. Do not capture inside an unfinished state.
- **Content:** use sample text only. No real names, no real locations, no actual reflections you would not publish.
- **Browser chrome:** use a clean profile or hide the bookmarks bar. No personal tabs visible.
- **Filename:** `screenshots/<flow>-<short-label>.png` (for example `screenshots/home-default.png`).

## Where to use them

- **Devpost gallery:** Home, Offshift, Moral Injury (3 required by submission).
- **README:** optional inline, but prefer linking to a `screenshots/` folder so the README stays light.
- **Demo recording:** see [`demo.md`](./demo.md) for the live-walkthrough order.
